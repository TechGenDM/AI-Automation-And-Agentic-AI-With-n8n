# Module 16: Multi Agent Architecture - Chain of Agents

> Status: ✅ Complete

---

## Quick Recap

This is the **first of five multi-agent architecture patterns**. Chain of Agents (also called **Prompt Chaining**) is the simplest one — a **relay race** where the task passes through multiple agents, one after another, each doing one specific job before handing off to the next.

---

## Core Concept

```
Input/Trigger → Agent A → Agent B → Agent C → Output
```

Each agent:
- Has its **own brain** (can literally be a different AI model — one agent on Claude, another on OpenAI, another on Gemini)
- Has its **own tools, memory, and instructions**
- Does **ONE specific job**, then passes its output to the next agent in line

> **No loops here** — agents fire strictly one after another, in a straight line. Just like a real-life chain of approvals: your manager approves first, then their manager, then finally HR — one after another, in sequence.

---

## Worked Example: Content Writing Pipeline

| Agent | Role | Job |
|---|---|---|
| **Agent A** | Researcher | Creates the initial raw content/idea — first draft based on user input |
| **Agent B** | Writer / Improver | Takes Agent A's output and improves it — grammar, structure, coherence, added depth |
| **Agent C** | Reviewer / Editor | Finalizes it — distills and refines into a polished final piece |

**Real build shown**: Researcher (with a search tool, e.g. Tavily) → Writer → Reviewer/Editor → final draft sent to Gmail or Slack. You can even add a **Human in the Loop** step here — the workflow pauses and waits for your approval before finalizing.

### Different brains for different agents
- Researcher & Reviewer → powered by **OpenAI (GPT-4o-mini)**
- Writer → powered by **Gemini Flash**

This is a direct real-world example of the advantage from the last module: use whichever AI model is genuinely best suited for each specific job, instead of being locked into one model for everything.

## Why Use Chain of Agents? (Benefits)

| Benefit | What it means |
|---|---|
| **Separation of Concerns** | Each agent focuses on ONE responsibility — much more reliable than asking one agent to "research, write, review, AND correct" all at once |
| **Modular** | You can improve or replace just ONE agent in the chain without rewriting the whole thing (e.g., swap in a better research tool without touching the writer or reviewer) |
| **Human-like Collaboration** | Mimics how real editorial/research teams actually work — draft → edit → review |
| **Better Results, Fewer Mistakes** | Each agent has a clear, narrow task, so it's less likely to get confused or produce a messy output |
| **More Control Per Step** | If one step (e.g., research) isn't good enough, you can fix just that piece — the rest of the chain stays untouched |
| **Easier to Debug** | If something breaks, it's immediately clear WHICH agent broke — instead of hunting through one giant agent with 15+ tools |
| **Reusable & Flexible** | Since each agent is a clean, focused unit, you can reuse individual agents in other workflows too |

> **One-line summary**: *"Chain of Agents turns one big task into a series of smaller, focused tasks — like assembling your own AI-powered team where each member knows exactly their job."*

## Important Naming Note

This pattern is also called **"Prompt Chaining"** in some frameworks/resources. Both names refer to the exact same thing — the task simply flows from one prompt/agent to the next.

> When discussing architecture with someone else, it's a good habit to briefly describe the diagram/flow too — since naming isn't 100% standardized across different people, frameworks, or resources.

---

## Practical Tip: Watching the Logs

When testing a Chain of Agents workflow, you can click into **each individual agent's execution** to see its own log:
- Which tool it called (e.g., a Google/Tavily search)
- What the model's input and output were
- How many **tokens** were used

This per-agent visibility is a real practical advantage over a single giant agent — debugging becomes much easier because you can see exactly what each specialized agent did, step by step, instead of one big black box.

---

## Key Takeaway

- **Chain of Agents = a relay race.** Task flows through Agent A → Agent B → Agent C, one after another, no loops.
- Each agent can have its **own brain, tools, and instructions** — mix and match the best AI model for each specific job
- Main advantage: **separation of concerns** — narrow, focused agents produce better, more reliable results than one agent trying to do everything
- Also called **"Prompt Chaining"** in other frameworks — same concept, different name
- Debugging is much easier — you can see exactly which agent (and which tool call) succeeded or failed, with full token/log visibility
