## Swarm Architecture In Depth

The module starts by reinforcing the big-picture architecture.

The instructor explains that the architecture should be drawn first, then built on the canvas.

If you do not need a particular agent, do not add it.
If you need a sequence of agents, wire them sequentially.
If you need hierarchy, build it hierarchically.

This is the practical rule for designing multi-agent systems in n8n.

## Master Workflow Structure

The master workflow is the top-level orchestrator.

The example includes:

- Telegram trigger as the input entry point

## Normalizing Inputs to Text

The lesson emphasizes a key implementation detail:

All inputs must ultimately become text before the agent can understand them.

That means:

- audio is transcribed to text
- images are analyzed and converted to text
- normal text is passed through as-is

This normalization is essential because the agent’s brain operates on text.

---

## Agent Collaboration and Tool Flow

The module describes how the assistant’s agents collaborate.

The flow is:

1. Telegram receives user input.
2. The input is normalized to final text.
3. The master agent examines the request.
4. The master agent decides which sub-agent(s) to call.
5. The chosen sub-agent executes its task.
6. The result flows back to the master agent.
7. The master agent sends the response back through Telegram or UI.

This is the end-to-end swarm execution loop.

---

## Fixing Triggers in Sub-Agents

A critical technical detail is standardizing trigger payloads.

Each sub-agent should accept the same input schema, typically `chatInput`.

Why this matters:

- it makes sub-agents reusable
- it simplifies master workflow calls
- it avoids mapping errors during orchestration

The module shows how to make all sub-agent triggers use the same field name so the master workflow can call them consistently.

## Supporting Voice Control

An advanced feature in this module is voice control.

Telegram voice messages are transcribed into text before reaching the agent.

This means the assistant can be controlled by speech as well as typing.

The workflow uses the same normalization principle:

- audio input → transcription → final text
- final text → agent processing

Voice support makes the assistant feel more natural and accessible.
